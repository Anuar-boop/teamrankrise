Website Speed Optimization: Why Slow Sites
Lose Customers (and Rankings)

By Alex · February 17, 2026 · 10 min read

Every second your website takes to load costs you money. A 1-second delay reduces conversions by 7%. A 3-second delay? 53% of mobile visitors leave before your page even finishes loading.

For NH businesses competing for local customers, a slow website is like having a store with a locked front door. People arrive, wait, and leave.

This guide shows you how to test your speed, what the numbers mean, and the specific fixes that make the biggest impact — ordered by effort vs. payoff.

How Speed Affects Your Business

Rankings

Google has used page speed as a ranking factor since 2010 (desktop) and 2018 (mobile). In 2026, Core Web Vitals are a confirmed ranking signal. Slow sites rank lower. Period.

Conversions

Speed directly impacts whether visitors become customers:

  • Pages loading in 1 second have a 3.05% conversion rate
  • Pages loading in 3 seconds: 1.61% (47% worse)
  • Pages loading in 5 seconds: 0.58% (81% worse)

For a Manchester business getting 500 organic visits/month, the difference between a 1-second and 3-second load time is roughly 7 extra leads per month. That adds up.

Bounce Rate

When your site takes too long, visitors leave without doing anything:

  • 1-second load: 7% bounce rate
  • 3-second load: 32% bounce rate
  • 5-second load: 90% bounce rate

High bounce rates tell Google your site isn’t satisfying users, which hurts rankings further. It’s a vicious cycle.

How to Test Your Speed (Free)

Google PageSpeed Insights (Best Starting Point)

Go to pagespeed.web.dev and enter your URL. You’ll get:

  • Performance score (0-100). Aim for 90+ on desktop, 70+ on mobile.
  • Core Web Vitals pass/fail for real user data
  • Specific recommendations with estimated time savings

GTmetrix (More Detail)

GTmetrix gives you a waterfall chart showing exactly what loads, in what order, and how long each element takes. It’s more technical but invaluable for diagnosing issues.

Google Search Console (Real User Data)

If you have Search Console set up, check Core Web Vitals under Experience. This shows how real visitors experience your site, not lab tests.

Understanding Core Web Vitals

Google measures three specific things:

LCP (Largest Contentful Paint): Under 2.5 Seconds

How long it takes for the biggest visible element (usually a hero image or heading) to appear. This is what users perceive as “the page loaded.”

INP (Interaction to Next Paint): Under 200 Milliseconds

How quickly your page responds when someone taps a button, clicks a link, or interacts. Replaced FID in March 2024.

CLS (Cumulative Layout Shift): Under 0.1

How much the page “jumps around” while loading. Ever tried to tap a button and the page shifted, making you tap an ad instead? That’s CLS.

The High-Impact Speed Fixes (In Priority Order)

1. Optimize Images (Biggest Win)

Images are typically 50-80% of a page’s total size. This is almost always the #1 speed issue for small business websites.

What to do:

  • Use WebP format instead of JPEG/PNG. WebP is 25-35% smaller with the same quality.
  • Resize images to display size. Don’t upload a 4000x3000 photo when it displays at 800x600. Resize before uploading.
  • Compress aggressively. Use tools like Squoosh (free, browser-based) or ShortPixel. Most images can be compressed 60-80% without visible quality loss.
  • Add lazy loading. Images below the fold should load only when users scroll to them. Add loading="lazy" to img tags.
  • Set explicit dimensions. Add width and height attributes to prevent layout shift (CLS).

Impact: Often cuts page size by 50-70% and shaves 1-3 seconds off load time.

2. Upgrade Hosting

Cheap shared hosting ($3-5/month) puts your site on a server with hundreds of other websites. When another site gets traffic, yours slows down.

What to do:

  • Minimum: SiteGround, Cloudways, or A2 Hosting ($15-30/month). Significantly faster than GoDaddy or Bluehost basic plans.
  • Better: Managed WordPress hosting like WP Engine or Kinsta ($30-50/month). Built-in caching, CDN, and optimization.
  • Best for simple sites: Static hosting on Netlify or Cloudflare Pages (free). If your site doesn’t need a CMS, this is the fastest option available.

Impact: Server response time (TTFB) improvement of 200-800ms. This affects every single page load.

3. Enable Caching

Caching stores copies of your pages so they don’t need to be rebuilt from scratch every time someone visits.

What to do:

  • Browser caching: Tell browsers to store CSS, JS, and images locally. Returning visitors load instantly.
  • Server-side caching: If using WordPress, install WP Rocket ($49/year) or the free LiteSpeed Cache plugin.
  • CDN (Content Delivery Network): Cloudflare (free plan) serves your site from servers worldwide. Visitors in New Hampshire get served from a nearby data center instead of wherever your hosting server is.

Impact: 40-70% faster page loads for repeat visitors. CDN reduces load time for all visitors.

4. Minimize and Defer JavaScript

JavaScript is the biggest render-blocking resource on most websites. It stops the page from displaying until it finishes loading.

What to do:

  • Remove unused JS. Many WordPress themes load 10+ scripts you don’t need. Dequeue them.
  • Defer non-critical JS. Add defer attribute to script tags so they load after the page renders.
  • Move analytics to the footer. Google Analytics, Facebook Pixel, etc. don’t need to load before your content.
  • Limit plugins. Each WordPress plugin potentially adds JS. Audit and remove what you don’t need.

Impact: 0.5-2 second improvement in perceived load time.

5. Optimize CSS Delivery

CSS blocks rendering too. The browser won’t display anything until it finishes parsing your stylesheets.

What to do:

  • Inline critical CSS. Put the CSS needed for above-the-fold content directly in the HTML <head>.
  • Minify CSS. Remove whitespace and comments. Tools: CSSNano, WP Rocket (does this automatically).
  • Remove unused CSS. Most themes include styling for features you don’t use. Tools: PurgeCSS, UnCSS.

6. Reduce Server Requests

Every image, script, stylesheet, and font is a separate request to the server. More requests = slower loading.

What to do:

  • Combine files. Merge multiple CSS files into one. Same for JS.
  • Use system fonts or limit web fonts. Loading 5 font weights from Google Fonts adds 300-500ms. Load only what you actually use.
  • Remove third-party scripts. Chat widgets, social sharing buttons, and marketing pixels add 1-3 seconds. Remove or defer everything that isn’t essential.

7. Fix Layout Shift (CLS)

Layout shift doesn’t affect speed per se, but it’s a Core Web Vital and affects user experience.

Common causes and fixes:

  • Images without dimensions: Always set width and height on <img> tags
  • Ads/embeds without reserved space: Set a min-height on containers
  • Web fonts causing text reflow: Use font-display: swap in your @font-face
  • Dynamic content injected above existing content: Insert below, not above

Speed by Website Platform

WordPress (Most NH Small Businesses)

WordPress can be fast, but most WordPress sites are slow because of theme bloat and too many plugins.

  • Use a lightweight theme (GeneratePress, Astra, Kadence) instead of Divi or Avada
  • Limit to 10-15 active plugins maximum
  • Install WP Rocket + Cloudflare
  • Use a quality host (SiteGround, Cloudways, WP Engine)

Squarespace / Wix / Weebly

Website builders are convenient but limited in speed optimization. You can’t access server settings or install caching plugins.

  • Optimize images before uploading (the builder won’t do it for you)
  • Limit custom code and third-party embeds
  • Use the builder’s built-in SEO settings
  • Accept that you’ll hit a speed ceiling — if speed matters, consider migrating to WordPress or a custom solution

Custom/Static Sites

Hand-coded HTML/CSS sites are naturally the fastest. If you’re using one:

  • Host on Cloudflare Pages, Netlify, or Vercel (free, globally distributed)
  • Optimize images and use WebP
  • Minify CSS/JS
  • You should be hitting 95+ on PageSpeed Insights easily

Quick Win Checklist

If you do nothing else, do these five things today:

  1. Run PageSpeed Insights on your homepage and top service page. Note your scores.
  2. Compress your images. Upload them to Squoosh.app and re-export as WebP. Replace on your site.
  3. Add Cloudflare free tier. 5-minute setup, instant CDN and basic caching.
  4. Check your mobile experience. Load your site on your phone. If it takes more than 3 seconds, you have a problem.
  5. Remove unused plugins/scripts. Deactivate anything you’re not actively using.

What “Fast Enough” Looks Like

For an NH local business website, these targets are realistic and sufficient:

  • PageSpeed desktop score: 85+
  • PageSpeed mobile score: 65+ (mobile is always lower)
  • LCP: Under 2.5 seconds
  • Total page size: Under 2MB (ideally under 1MB)
  • Time to first byte: Under 600ms
  • Full load time: Under 3 seconds on 4G mobile

You don’t need a perfect 100 score. You need to be faster than your competitors. In most NH markets, even basic speed optimization puts you ahead of 80% of local business sites.
